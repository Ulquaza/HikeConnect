# 🏔️ HikeConnect

**HikeConnect** — это сервис подбора напарников для походов (в первую очередь горных), который помогает находить не просто попутчиков, а **совместимых людей по стилю, темпу и отношению к походу**.

Проект решает проблему случайных и часто неудачных подборов группы, когда совпадают даты, но не совпадают люди.



## 🚀 Идея

В отличие от обычных сервисов поиска попутчиков, HikeConnect делает акцент не только на маршруте, но и на **человеческой совместимости**.

Пользователь:
- создаёт профиль походника,
- описывает свой опыт и стиль,
- проходит поведенческий опрос,

а система:
- анализирует его характеристики,
- показывает **процент совместимости** с другими людьми,
- помогает собрать более «сработанную» команду.



## 🎯 Проблема

Сейчас поиск напарников происходит через:
- чаты,
- форумы,
- знакомых,
- случайные группы.

Это создаёт ряд рисков:
- ❌ неизвестный уровень подготовки участников  
- ❌ разный темп и стиль прохождения  
- ❌ бытовая и психологическая несовместимость  
- ❌ отсутствие проверки репутации  
- ❌ срывы договорённостей  

👉 В горах это влияет не только на комфорт, но и на безопасность.



## 💡 Решение

HikeConnect предлагает:

- 📊 **Поведенческий профиль** (отношение к риску, темп, дисциплина)
- 🤝 **Подбор по совместимости**
- ⭐ **Система отзывов и репутации**
- 🧭 **Создание и поиск походов**
- 🧑‍🤝‍🧑 **Контроль состава группы**

Сервис **не является туроператором** и не участвует в финансовых операциях — он только помогает людям находить друг друга.



## 👤 Целевая аудитория

- Возраст: 18–50 лет  
- Люди с опытом походов (особенно горных)  
- Нет постоянной команды  
- Ценят:
  - предсказуемость,
  - ответственность,
  - совместимость по стилю

Типичный пользователь — человек с рациональным мышлением, который не хочет идти в поход «вслепую».



## ⚙️ Основной функционал (MVP)

- Регистрация и профиль пользователя  
- Поведенческий опрос  
- Расчёт процента совместимости  
- Просмотр профилей других пользователей  
- Создание походов  
- Подача заявок на участие  
- Одобрение/отклонение участников  
- Отзывы после походов  



## 🏗️ Архитектура

Проект построен по принципам **Clean Architecture**:

- **MountLib.Core** — сущности и интерфейсы  
- **MountLib.Application** — бизнес-логика  
- **MountLib.Infrastructure** — работа с БД  
- **MountLib.Api** — Web API  
- **MountLib.BlazorApp** — клиентское приложение  



## 🛠️ Технологии

- **C#**
- **Blazor WebAssembly**
- **ASP.NET Core Web API**
- **PostgreSQL**
- **Docker**



## 📌 Особенности

- Упор на **человеческий фактор**, а не только на маршрут  
- Снижение конфликтов в походах  
- Формирование культуры осознанного подбора команды  
- Накопление репутации пользователей со временем  



## ⚠️ Ограничения

- Сервис не гарантирует безопасность  
- Не заменяет опыт и здравый смысл  
- Не участвует в организации походов  



## 🔮 Планы развития

- Улучшение алгоритма совместимости  
- Более точная модель поведения пользователя  
- Расширенная система репутации  
- Уведомления и автоматизация взаимодействия
